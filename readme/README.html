Project Name: Sauce Labs

Description

This is a mobile automation project is designed to automate Sauce Labs application testing using Intellij. The purpose of this project is to provide a framework that facilitates efficient and reliable testing of mobile applications, allowing for faster identification of bugs and ensuring a high-quality user experience. By automating repetitive manual tasks, this project enables testers to focus on more complex scenarios and edge cases.

Usage

  1. Clone the Repository: Start by cloning the project repository to your local machine using the following command:
    git clone https://github.com/mohamedyussif/SauceLabswithShaftFramework.git
  2. Setup appium inspector: Open appium inspector and set the desired capabilities for application inspection following command:
    { "platformName": "Android", "appium:deviceName": "Demo", "appium:app": "<Your apk directory>\\Android.SauceLabs.Mobile.Sample.app.2.2.0.apk", "appium:appWaitActivity": "com.swaglabsmobileapp.MainActivity", "appium:platformVersion": "14.0", "appium:automationName": "UiAutomator2" }
  3. Configure Device and Environment: Set up the desired mobile device and environment configurations in the relevant configuration files. Make sure to specify the appropriate device capabilities, platform versions, and other desired settings.
  4. Open the project: Navigate to the poject and open it from intellij.
  5. Run the project: Wait for loading the dependencies then run the project from the project terminal by using the following command.mvn test -Dtest="Test_SauceLabsWithShaft"
  6. Reporting: Please wait after executing the project for extracting the execution summary report and the allure report

Code Styling

The project follows a Page Object Model Design to maintain readability and code quality. The following guidelines should be considered:

  • Naming Conventions: Follow camel case naming conventions for variables, and functions. Use descriptive and meaningful names to enhance code comprehension.
  • Comments: Document the code using inline comments to provide context and explanations where necessary. Commenting is encouraged for complex logic or non-obvious functionality.
  • Modularity and Reusability: Promote modularity and reusability by encapsulating every needed locators or functions in a reusable methods to Aim for single responsibility and avoid code duplication.
  • Error Handling: Implement proper error handling mechanisms to gracefully handle exceptions and avoid unexpected applicationcrashes.
